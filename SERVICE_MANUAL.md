# KTALK TALK 서비스 매뉴얼

> 어색한 분위기를 깨는 가장 짜릿한 대화 게임

---

## 1. 서비스 개요

KTALK TALK은 **그룹 코칭 대화 게임**입니다. 아케이드 게임 테마의 인터페이스로 팀 빌딩, 회고, 리더십 코칭 등 다양한 상황에서 구조화된 대화를 이끌어냅니다.

| 항목 | 내용 |
|------|------|
| **대상 사용자** | 그룹 코칭 퍼실리테이터, 팀 리더, HR 담당자 |
| **참여 인원** | 2~7명 |
| **기술 스택** | HTML/CSS/JS 단일 파일 SPA, Supabase, QRCode.js |
| **배포** | Vercel (GitHub 연동 자동 배포) |

## 2. 화면 구성 및 흐름

### 전체 흐름도

```
HOME → SETUP(인원) → NAMES(이름) → TOPIC(주제) → STAGING(대기) ↔ GACHA(질문카드)
                                                                       ↓
                                                                   SUMMARY(결과)
                                                                       ↓
                                                              COMMIT(다짐 폼, QR)
```

### 2.1 HOME (홈)
- 서비스 소개 히어로 영역
- 주요 기능 안내 카드
- 주제 태그 미리보기
- **[게임 시작]** 버튼 → SETUP 이동

### 2.2 SETUP (인원 설정)
- 2~7명 중 참여 인원 선택
- 사람 아이콘으로 시각적 표시
- **[NEXT]** → NAMES 이동

### 2.3 NAMES (이름 입력)
- 참여자 이름 입력 폼
- 미입력 시 기본값: "플레이어 1", "플레이어 2" 등
- **[START]** → TOPIC 이동

### 2.4 TOPIC (주제 선택)
- 6개 카테고리 카드에서 선택
- 대화 깊이(Depth) 레벨 선택 (1~3단계)
- 선택 후 슬롯머신 애니메이션과 함께 STAGING 진입

### 2.5 STAGING (대화 대기)
- 💬 대화 진행 중 비주얼 (녹음 우려 없는 말풍선 아이콘)
- 참여자 발언 추적 버튼 (설정 ON일 때만 표시)
- 주제 변경 가능
- 넛지 메시지 자동 표시 (60초 간격)
- 세션 시간은 내부적으로 기록되나 화면에는 표시되지 않음
- **[카드 뽑기]** → 슬롯머신 → GACHA

### 2.6 GACHA (질문 카드)
- 뽑힌 질문 카드 표시 (질문 + 퍼실리테이터 팁)
- 주제 선택 변경 가능
- 순서 뽑기 (랜덤 참여자 선택 애니메이션)
- **[RE-DRAW]** 카드 재뽑기
- **[깊이+/깊이-]** 대화 깊이 조절
- **[BACK]** → STAGING 복귀
- **[세션 종료]** → SUMMARY

### 2.7 SUMMARY (세션 결과)
- **SESSION COMPLETE** + 경과 시간
- 배지: 주제, 깊이, 인원, 카드 수, 재뽑기 횟수
- **오늘 사용한 카드** 리캡 (카테고리 색상 구분)
- **세션 강도** 바 차트 (카드 사용, 대화 깊이, 재뽑기)
- 발언 추적 ON 시: 링 차트 + 참여자별 발언 비율
- 건강 점수는 참여자에게 숨겨져 있음 (관리자 히스토리에서만 확인)
- **QR 코드** 다짐 페이지 연결
- 실시간 제출 현황 표시 (✅ 칩 클릭 시 해당 참여자의 성찰·다짐 내용 팝업)

### 2.8 COMMIT (다짐 폼)
- QR 스캔 시 접근하는 참여자용 페이지
- 참여자 선택 드롭다운
- 오늘의 성찰(reflection) 텍스트 입력
- 실행 다짐(action item) 텍스트 입력
- 제출 후 자동으로 세션 건강 점수에 반영

### 2.9 ADMIN (관리자)
- 우측 상단 설정(⚙️) 아이콘으로 진입
- 비밀번호 인증 후 관리 기능 접근

## 4. 대화 깊이 시스템

| 레벨 | 표시 | 설명 |
|------|------|------|
| 1단계 | 🟢 가볍게 | 편안한 대화, 아이스브레이킹 |
| 2단계 | 🔵 깊게 | 중간 수준의 성찰적 질문 |
| 3단계 | 🟣 솔직하게 | 깊은 개인적 이야기, 높은 심리적 안전 필요 |

- 질문은 설정된 깊이 **이하** 모든 레벨에서 출제
- 세션 중 깊이+/깊이- 버튼으로 실시간 조절 가능

---

## 5. 꽝(Bbang) 시스템

- 카드 뽑기 시 설정된 확률(기본 5%, 0~30%)로 꽝 카드 출현
- 꽝 카드는 미니 미션 또는 유머 메시지
- 예: "5초 침묵해보세요. 침묵도 코칭입니다."
- 확률은 관리자 패널에서 슬라이더로 조절

---

## 6. 슬롯머신 애니메이션

- 3개 릴이 순차적으로 멈추는 아케이드 스타일
- 릴 0: 1.5초, 릴 1: 2.0초, 릴 2: 2.5초에 정지
- 마지막 심볼은 선택된 주제의 이모지
- "MATCHED!" 표시 후 질문 카드 화면 전환

---

## 7. 순서 뽑기 (랜덤 참여자 선택)

GACHA 화면에서 **[🎲 순서 뽑기]** 버튼으로 사용:

1. 버튼 클릭 → 미선택 참가자 카드가 빠르게 랜덤 하이라이트 (15회, 100ms 간격)
2. 애니메이션 종료 → 1명 선택, 핑크 글로우 효과
3. 선택된 참가자에 순번 뱃지 표시
4. 다시 클릭 → 나머지 중 다음 선택
5. 전원 선택 완료 → "순서 초기화" 버튼으로 전환
6. RE-DRAW 시 순서 자동 초기화

## 8. 발언 추적 시스템

> 관리자 설정에서 ON/OFF 가능 (기본: OFF)

### 수동 추적
- 참여자 이름 버튼 클릭 또는 숫자키(1~7) → 발언 시작/종료 토글
- 발언 중인 참가자: 핑크 글로우 펄스 효과
- 1초 간격으로 발언 시간 누적

### 마이크 감지
- 관리자 설정에서 별도 ON/OFF (기본: OFF)
- Web Audio API로 주변 소리 감지
- 60초 이상 무음 시 넛지 메시지 자동 표시

## 9. 넛지(Nudge) 시스템

60초 간격으로 자동 표시되는 코칭 메시지 (총 30개):

| 유형 | 조건 | 메시지 수 | 색상 |
|------|------|-----------|------|
| **아이들(Idle)** | 활성 발언자 없이 60초 경과 | 10개 | 시안 |
| **밸런스(Balance)** | 발언 시간 편차 60초 이상 | 10개 | 핑크 |
| **사일런스(Silence)** | 일반 코칭 팁 | 10개 | - |
| **마이크(Mic)** | 마이크 무음 60초 이상 | 4개 | - |

넛지 메시지는 6초간 표시 후 자동 사라짐.

## 10. 세션 건강 점수

세션 종료 시 자동 계산되어 DB에 저장 (관리자 히스토리에서 확인 가능):

| 지표 | 가중치 | 계산 방식 |
|------|--------|-----------|
| **균형(Balance)** | 35% | 참여자 발언 시간의 변동계수 기반 |
| **깊이(Depth)** | 20% | 1단계:33 / 2단계:66 / 3단계:100 |
| **밀도(Density)** | 20% | 총 발언 시간 / 세션 시간 비율 |
| **참여(Coverage)** | 15% | 발언한 참여자 비율 |
| **다짐(Commitment)** | 10% | QR 다짐 제출 비율 |

### 등급 표시

| 점수 구간 | 등급 |
|-----------|------|
| 85점 이상 | 🌟 완벽한 대화였어요! |
| 70~84점 | ✨ 훌륭한 시간이었어요! |
| 55~69점 | 💪 좋은 대화를 나눴어요! |
| 40~54점 | 🌱 함께 성장하고 있어요! |
| 40점 미만 | 🔥 좋은 시작이에요! |

> 세션 종료 화면에서는 점수가 숨겨져 있으며, 관리자 패널의 세션 히스토리에서만 확인 가능합니다.

## 11. QR 다짐(Commitment) 시스템

1. 세션 종료 → Supabase에 세션 저장 → QR 코드 자동 생성
2. QR 코드에 인코딩된 URL: `{사이트주소}#commit/{세션ID}`
3. 참여자가 핸드폰으로 스캔 → 다짐 입력 폼 접근
4. 성찰 + 실행 다짐 작성 후 제출
5. 세션 결과 화면에서 5초마다 제출 현황 자동 갱신
6. 제출 시 건강 점수의 다짐(Commitment) 지표 업데이트
7. 제출 완료된 참여자 칩(✅)을 클릭하면 **팝업으로 성찰·다짐 내용 확인** 가능
   - 팝업 닫기: 우상단 X 버튼 또는 팝업 바깥 영역 클릭

## 12. 관리자 기능

> PC 화면에 최적화된 넓은 레이아웃 (768px 이상에서 800px 폭)

### 12.1 접근 방법
- 화면 상단 설정(⚙️) 아이콘 클릭
- 비밀번호 입력: `ktalk2024!`
- 로그인 후 관리 콘텐츠 표시

### 12.2 설정 관리
| 설정 | 기본값 | 설명 |
|------|--------|------|
| 발언자 추적 | OFF | 참여자 발언 시간 추적 UI 표시 |
| 마이크 감지 | OFF | 마이크 무음 감지 + 넛지 |

### 12.3 질문 카드 관리
- **카테고리 탭**: 워밍업, Jira·실행, 리더십, 동료, 회고, 꽝!
- **질문 추가**: 카테고리, 질문 텍스트, 퍼실리테이터 팁, 깊이 레벨
- **질문 수정**: 인라인 편집 폼
- **질문 삭제**: 확인 대화상자 후 삭제
- **꽝 확률 조절**: 슬라이더 (0~30%)
- **깊이별 그룹핑**: 카테고리 내 1/2/3단계 헤더로 구분

### 12.4 세션 히스토리
- **점수 추세 차트**: 최근 세션들의 건강 점수 바 차트
- **세션 목록**: 날짜, 주제, 깊이, 시간, 인원, 점수
- **세션 상세**: 건강 점수 분해, 참여자별 발언 시간, 다짐 기록
- **세션 삭제**: 연관 데이터(다짐, 노트) 자동 삭제 (CASCADE)

### 12.5 관리자 메모
- 세션 상세에서 메모 추가/수정/삭제
- Google Form 결과 등 외부 데이터 기록용
- 다른 관리자와 정보 공유 목적

## 16. 세션 운영 가이드

### 사전 준비
1. 관리자 로그인 → 질문 카드 확인/편집
2. 발언 추적/마이크 감지 필요 시 설정 ON
3. 꽝 확률 적절히 조정 (아이스브레이킹 목적 시 높게)

### 세션 진행
1. HOME → 게임 시작 → 인원 설정 → 이름 입력
2. 주제 선택 + 대화 깊이 설정
3. STAGING에서 [카드 뽑기] → 질문에 따라 대화
4. 필요 시 순서 뽑기로 발언 순서 결정
5. 깊이 조절, 주제 변경 자유롭게 활용
6. 충분히 대화 후 [세션 종료]

### 세션 마무리
1. SUMMARY 화면에서 카드 리캡, 세션 강도 확인
2. QR 코드를 참여자에게 보여줌
3. 참여자들이 핸드폰으로 스캔하여 다짐 작성
4. 제출 현황 실시간 확인
5. 관리자 히스토리에서 세션 메모 추가

## 17. 문제 해결

| 증상 | 원인 | 해결 |
|------|------|------|
| QR 코드 안 보임 | CDN 로드 실패 또는 Supabase 연결 실패 | 네트워크 확인, 페이지 새로고침 |
| 질문이 안 나옴 | Supabase 연결 실패 | 콘솔에서 에러 확인, 네트워크 상태 점검 |
| 마이크 권한 요청 | 마이크 감지 ON 상태 | 관리자에서 OFF로 변경하거나 권한 허용 |
| 발언 추적 안 보임 | 기본 OFF 설정 | 관리자 → 설정 → 발언자 추적 ON |
| 세션 히스토리 비어있음 | 세션 완료 기록 없음 | 세션을 끝까지 진행하여 종료 |
| 세션 삭제 안 됨 | DB 권한 정책(RLS) 누락 | Supabase에서 DELETE 정책 확인 |
